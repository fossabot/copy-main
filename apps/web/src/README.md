# توثيق مجلد frontend/src

## 1. نظرة عامة (Overview)
هذا المجلد يحتوي على الكود المصدري للواجهة الأمامية (Frontend) لمنصة "النسخة". تم بناء الواجهة باستخدام **Next.js 16** مع **TypeScript** و **Tailwind CSS**، مع التركيز على دعم اللغة العربية (RTL) وتوفير تجربة مستخدم سينمائية تفاعلية.

## 2. البنية المعمارية (Architecture)
يعتمد التطبيق على **Next.js App Router**:
- **App Layer (`app/`)**: تحتوي على الصفحات، التخطيطات (Layouts)، ومسارات API.
- **Components Layer (`components/`)**: مكونات واجهة المستخدم مقسمة حسب الوظيفة (UI, Auth, Editors).
- **AI Layer (`ai/`)**: خدمات التكامل مع Gemini AI في جهة العميل.
- **State Management (`lib/stores/` & `hooks/`)**: إدارة حالة التطبيق باستخدام **Zustand** و **React Query**.
- **Workers Layer (`workers/`)**: معالجة المهام الثقيلة (مثل تأثيرات الجسيمات Particle Systems) في الخلفية لضمان سلاسة الواجهة.

## 3. المكونات الرئيسية

### 3.1 صفحات التطبيق (`src/app/`)
- `layout.tsx`: التخطيط الرئيسي مع دعم RTL وخط Cairo.
- `(main)/ui/`: مشغل التطبيقات الـ 13.
- `(main)/breakdown/`: واجهة تحليل السيناريو.
- `(main)/editor/`: محرر السيناريو الاحترافي.

### 3.2 واجهة المستخدم الذكية (`src/components/`)
- `seven-stations-interface.tsx`: الواجهة الرئيسية لخط الإنتاج السينمائي (Seven Stations).
- `particle-background-optimized.tsx`: نظام الجسيمات التفاعلي للخلفية.
- `ErrorBoundary.tsx`: معالجة الأخطاء لضمان استقرار التطبيق.

### 3.3 خدمات الذكاء الاصطناعي (`src/ai/`)
- `gemini-service.ts`: خدمة التواصل مع نماذج Gemini.
- `ai-team-brainstorming.ts`: منطق العصف الذهني الجماعي للوكلاء.

## 4. تدفق البيانات (Data Flow)
1. يتفاعل المستخدم مع المكونات (`Components`).
2. يتم تحديث الحالة المحلية أو العالمية عبر `Stores` (Zustand).
3. تُرسل الطلبات إلى الخلفية (Backend) أو مباشرة إلى `Gemini API` عبر `Services`.
4. تُعرض النتائج (تقارير، تحليلات، مشاهد مولدة) للمستخدم بشكل لحظي.

## 5. الأداء وتجربة المستخدم
- **Streaming**: استخدام خاصية التوجيه المتدفق في Next.js.
- **Web Workers**: تشغيل محركات الفيزياء والجسيمات بعيداً عن الخيط الرئيسي.
- **ISR**: تحديث الصفحات بشكل ثابت وتدريجي لضمان السرعة.
- **Device Detection**: تخصيص التجربة بناءً على نوع الجهاز وقدراته.
